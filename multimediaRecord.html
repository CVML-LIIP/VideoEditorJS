Enregistrement audio et vidéo

L'enregistrement audio/vidéo via le navigateur s'est considérablement simplifié depuis l'arrivée de HTML5.
Clangue.net étant un projet ayant pour principale vocation l'enregistrement en ligne des élèves, cette partie fut relativement simple.

le Principe de base a été complètement importé depuis clangue.net, qui permettait l'enregistrement audio uniquement.

Mais depuis le temps une communauté de développeur ont créé un projet utilisant les technologies html5, pour simplifier le portage des méthodes sur tous les navigateurs. Ces développeurs ont donc conçu RecordRTC ( lien du projet), qui vise à simplifier cette tâche et permettant l'enregistrement sur des formats natifs à html5 plus facilement, comme l'OGG pour l'audio, ou le WEBM pour la vidéo. Cependant, le projet n'est pas encore parfait et demande donc encore une adaptation aux différentes situations. Ainsi pour le navigateur Google Chrome, par exemple L'enregistrement vidéo ne peut se faire que par l'enregistrement de manière indépendante l'audio et la vidéo.
Ce qui demande donc un traitement avec FFmpeg.js pour combiner ces deux partie, avant d'insérer le fichier dans le projet et de lui faire subir les traitements habituels de n'importe quel fichier importé par l'utilisateur.

A propos

VideoEditorJs (abrégé VEJS), est un projet qui a pour but de permettre le montage vidéo a ceux qui qui n'ont pas les logiciel necessaire, ni les compétence. VEJS, devrait à terme être associé a site clangue, dans le but d'y proposer l'enregistrement vidéo en plus de l'enregistrement audio. Ce projet, est aussi conçus dans l'optique qu'il devienne un module utilisable facilement sur d'autres sites.
